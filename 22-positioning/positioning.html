Positioning

  1. The default value for position is static
  2. Some of the properties you need to supply when using any non-deault position value are: top, left, bottom, and right.
  3. The inferred display value of an element has been positioned is: the value display: block.
  4. The thing that is special about fixed and sticky positioning is they always creat a new Stacking context.
    Fixed positioning always keeps the navigation present at all time on the page. Position fixed doesn't move as scroll on the viewport.
    Sitckcy positioning allows us to make elements stick when the scroll reaches a certain point.
  5. One of the non-default values for position still maintains a box in the flow document, that is relative
  6. Absolute positioned layouts are useful for elements that should not interact. It doesn' t affect any elements in the document.
  7. When positioning element absolutely, to lock that element' s coordinates to its parent element is set the properties top, let, right, and bottom.
  8. Sometimes if you put two absolutely positioned elements in the same physical space, they' ll stack, that means only one will be visible on top. The rules here are:
    • Controling the stacking order by using Z-index on positioning elements.
    • Opacity might by used at some cases.
    A certain is element on top because when Z-index propertie is not specifed on any element, elements are stacked in the following order (from bottom to top)
  9. Css' s Stacking Content is a three dimensional conceptualization of html along an imaginary Z-axis relative to user. A stacking content is created when an element is positioned and assigned a z-index value other than auto.
