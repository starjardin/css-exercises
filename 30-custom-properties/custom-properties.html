Custom Properties

Research about css Custom Properties

  1. The characters that we have to use when defining a Custom Property' s name are the prefix double dashes (--), any alphanumerics characters, emojis , and underscore can be used as well.
  2. In Custom Properties name we can' t use upercase, comas, question mark, space, any characters apart from dashes, alphanumerics characters, underscore, and emojis.
  3. Possible values that we can assing to a new Custom Property: absolute values, variable values.
  4. I think it is useful to create Custom Property using the :root or html selector when you want to apply it to all of the elements in the document and the Property gets inherited. It isn' t useful when the elements don't get inherit or you just want to apply it in a specific block of an element.
  5. After we've defined a new Custom Property, we can recall or use that Custom Property in function. We can use the var function var()
  6. There are several ways to overrid or change a Custom Property:
    • use a new selector.
    • use a new name of the Custom Property.
    • modify the curent values
  7. Custom Properties are useful for modifying values, for relationships in the document. Clean and less code to write.
  8. The only aspect (or part) of CSS syntax that we can use Custom Properties for is the values
